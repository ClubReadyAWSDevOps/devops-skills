## AWS Comprehensive Audit

Master orchestrator that runs all AWS audit skills in parallel, consolidates findings, and creates a GitHub issue for tracking remediation.

### Prerequisites

- AWS CLI configured with read permissions across all services
- GitHub CLI (`gh`) authenticated
- Repository with Issues enabled
- `jq` installed for JSON parsing

### Workflow

1. **Spawn parallel audit subagents**
   
   Launch 4 independent audit agents (no blocking dependencies):
   - Cost review (`/aws-cost-review`)
   - Credentials audit (`/aws-credentials-audit`)
   - Architecture review (`/aws-architecture-review`)
   - Reserved capacity review (`/aws-reserved-capacity`)

2. **Wait for all audits to complete**
   
   Each subagent returns structured findings. Capture:
   - Cost review: Monthly spend, trends, anomalies, top drivers
   - Credentials: Unused keys, MFA gaps, old passwords, rotation status
   - Architecture: HA/security/cost/performance scores + top 5 action items
   - Reserved capacity: RI utilization, expirations, On-Demand gaps, savings potential

3. **Consolidate findings into priority matrix**
   
   Rank all findings by:
   - **Critical (P0):** Security gaps, publicly accessible resources, root account issues
   - **High (P1):** Cost waste >$1k/month, compliance violations, RI expirations <30 days
   - **Medium (P2):** Architecture improvements, cost optimization opportunities
   - **Low (P3):** Documentation, monitoring enhancements

4. **Calculate total impact**
   
   Aggregate metrics:
   - Total monthly cost: $X
   - Identified waste: $Y/month
   - Potential savings: $Z/month (RI optimization + rightsizing + cleanup)
   - Critical security issues: N
   - Action items: M

5. **Generate GitHub issue body**
   
   Use this markdown template:
   
   ```markdown
   # AWS Infrastructure Audit — [YYYY-MM-DD]
   
   **Executive Summary**
   - **Total Monthly Cost:** $X
   - **Identified Waste:** $Y/month
   - **Savings Opportunity:** $Z/month
   - **Critical Issues:** N 🔴
   - **Action Items:** M total
   
   ---
   
   ## 🔴 Critical Issues (P0)
   
   1. [credentials-audit] Root MFA disabled — enable immediately
   2. [architecture] RDS instance publicly accessible — remove public access
   
   ## 🟠 High Priority (P1)
   
   1. [cost] $1,200/month waste from 12 unattached EBS volumes — delete
   2. [reserved-capacity] db.r5.large RI expires 2026-07-10 (15 days) — renew or convert
   
   ## 🟡 Medium Priority (P2)
   
   1. [cost] $800/month potential savings from Graviton migration (3 m5 instances)
   2. [architecture] VPC Flow Logs not enabled — enable for security/compliance
   
   ## 🟢 Low Priority (P3)
   
   1. [architecture] 5 S3 buckets without access logging
   2. [credentials] 3 IAM users without MFA (non-production accounts)
   
   ---
   
   ## Detailed Findings
   
   <details>
   <summary>📊 Cost Review</summary>
   
   [Full cost review output from subagent]
   
   </details>
   
   <details>
   <summary>🔐 Credentials Audit</summary>
   
   [Full credentials audit output from subagent]
   
   </details>
   
   <details>
   <summary>🏗️ Architecture Review</summary>
   
   [Full architecture review output from subagent]
   
   </details>
   
   <details>
   <summary>💰 Reserved Capacity Review</summary>
   
   [Full RI/Savings Plans review output from subagent]
   
   </details>
   
   ---
   
   ## Next Steps
   
   - [ ] Address all P0 (Critical) issues within 24 hours
   - [ ] Review P1 (High) findings and assign owners
   - [ ] Evaluate P2 (Medium) cost optimization opportunities
   - [ ] Schedule P3 (Low) improvements for next sprint
   
   **Generated by:** [aws-comprehensive-audit](https://github.com/ClubReadyAWSDevOps/devops-skills)  
   **Audit Date:** YYYY-MM-DD HH:MM UTC
   ```

6. **Create GitHub issue**
   
   ```bash
   # Generate issue title
   ISSUE_TITLE="AWS Infrastructure Audit — $(date -u +%Y-%m-%d)"
   
   # Save consolidated report to temp file
   cat > /tmp/aws-audit-report.md <<'EOF'
   [Generated markdown from step 5]
   EOF
   
   # Create issue with labels
   gh issue create --repo ClubReadyAWSDevOps/devops-skills --title "$ISSUE_TITLE" --body-file /tmp/aws-audit-report.md --label "aws-audit,monthly-review" --label "priority:high"
   
   # Capture issue URL
   ISSUE_URL=$(gh issue list --repo ClubReadyAWSDevOps/devops-skills --limit 1 --json url --jq '.[0].url')
   
   echo "✅ AWS Audit complete"
   echo "📋 GitHub Issue: $ISSUE_URL"
   ```

7. **Post-audit cleanup**
   
   ```bash
   # Clean up temp files
   rm -f /tmp/aws-audit-report.md
   
   # Optional: Send Slack notification
   # curl -X POST -H 'Content-type: application/json' \
   #   --data '{"text":"AWS Audit complete: '"$ISSUE_URL"'"}' \
   #   $SLACK_WEBHOOK_URL
   ```

### Integration Pattern

This skill should be invoked as:

**Manual (via Claude Code):**
```bash
cd ~/projects/devops-skills
/aws-comprehensive-audit
```

**Scheduled (via GitHub Actions cron):**
```yaml
name: Monthly AWS Audit

on:
  schedule:
    # Run on 1st of every month at 9 AM UTC
    - cron: '0 9 1 * *'
  workflow_dispatch:  # Allow manual trigger

jobs:
  audit:
    runs-on: ubuntu-latest
    permissions:
      id-token: write  # OIDC token
      contents: read
      issues: write    # Create issue
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: arn:aws:iam::${{ secrets.AWS_ACCOUNT_ID }}:role/aws-audit-github-actions
          aws-region: us-west-2
      
      - name: Install dependencies
        run: |
          sudo apt-get update
          sudo apt-get install -y jq bc
      
      - name: Run comprehensive audit
        run: |
          # This would invoke the skill via a wrapper script
          # that calls each audit skill and consolidates output
          ./scripts/run-comprehensive-audit.sh
        env:
          GH_TOKEN: ${{ github.token }}
```

### Error Handling

If any subagent fails:

1. **Continue with remaining audits** — don't fail entire workflow
2. **Mark failed section in report** — add warning to GitHub issue:
   ```markdown
   ## ⚠️ Credentials Audit Failed
   
   Error: IAM credential report generation timed out
   
   Manual action required: Run `/aws-credentials-audit` separately
   ```
3. **Include partial results** — better than no audit at all

### Notes

- **Execution time:** ~3-5 minutes total (parallelized)
- **AWS API calls:** ~50-100 (read-only, safe to run frequently)
- **IAM permissions required:** Same as individual audit skills (read-only across all services)
- **Cost:** ~$0 (read-only API calls are free, no CloudWatch Insights queries)
- **GitHub issue repo:** Can be configured via env var `GITHUB_AUDIT_REPO`

### Customization

Override defaults via environment variables:

```bash
# Change target repository for issues
export GITHUB_AUDIT_REPO="ClubReadyAWSDevOps/infrastructure"

# Set Slack webhook for notifications
export SLACK_WEBHOOK_URL="https://hooks.slack.com/services/..."

# Skip specific audits
export SKIP_AUDITS="reserved-capacity"  # Comma-separated

# Run audit
/aws-comprehensive-audit
```

### Integration with Other Skills

- **After audit:** Use findings to trigger specific skills
  - High waste → `/aws-snapshot-cleanup`
  - Low CPU → `/aws-rds-rightsizing`
  - RI expiring → `/aws-reserved-capacity` (detailed)
- **Before changes:** Run comprehensive audit to establish baseline
- **Monthly cadence:** Scheduled via GitHub Actions cron

### Troubleshooting

**Issue creation fails:**
```bash
# Check GH_TOKEN is set
echo $GH_TOKEN

# Test gh CLI auth
gh auth status

# Verify repo has issues enabled
gh repo view ClubReadyAWSDevOps/devops-skills --json hasIssuesEnabled
```

**Subagent timeout:**
- Default timeout: 5 minutes per audit
- If architecture review exceeds this, run separately with higher timeout
- Partial results still published to issue

**Duplicate issues:**
- Check for existing open audit issues before creating
- Close previous month's issue when new one is created
- Or use single tracking issue with monthly updates via comments
